## 2. Core Architectural Components & Roles

### 2.1 The Episemantic Framework (UMB-ESF-001) - *The Legislative Branch*
If the knowledge graph is the structure, the Episemantic Framework is the "living context" flowing through it.
*   **Function:** It governs the application of **Episemantic Markers**—machine-readable contextual annotations that act as the "Kigo" (seasonal words) of a cell. 
*   **Synergy:** The ESF instructs the Coherent Synthesis Engine (CSE) on how to interpret information based on current systemic or relational states. It prevents static information from becoming obsolete by ensuring all data is anchored to its current contextual relevance.

### 2.2 The Truth Compass (UMB-TC-001) - *The Epistemic Sensor*
*   **Function:** Acts as the sensory interface for the Self-Integrity Validation Core (SIVC). It provides real-time "analog" epistemic confidence signals (e.g., True North vs. Dissonant Drift).
*   **Synergy:** It acts as a passive, continuous gauge. If the Episemantic Framework assigns a marker, the Truth Compass monitors the validity of that marker over time. If a linked artifact begins to contradict the established codex, the Truth Compass registers the variance and triggers the CRLPF's Anomaly Detector.

### 2.3 Active Validation (GUCA-WEAVE-VALIDATE-001) - *The "Tester"*
*   **Function:** A proactive defense mechanism. The Tester does not wait for failure; it actively simulates validation scenarios and stress-tests system integrity before deployment or state finalization.
*   **Synergy:** Working in tandem with the Truth Compass and the Meta-Engine, the Tester executes "dry runs" (e.g., the Cerberus adversarial tests). It ensures that the links built by the ALP and the context applied by the ESF hold up under extreme systemic pressure without causing recursive feedback loops.

### 2.4 Compliance Enforcement (GUCA-WEAVE-ALIGN-001) - *The "Judge"*
*   **Function:** The final arbiter of artifact canonization. The Judge audits artifacts for mandatory compliance markers: Prime Alignment, Traceability, and Persona Alignment.
*   **Synergy:** Once the Tester proves that a process works without breaking the system, the Judge proves that the output *belongs* in the system according to the Architect's Codex (`CODEX-PRIME-001`). If an artifact lacks an established Registry ID or violates the Law of One-Way Flow, the Judge vetoes its inclusion and triggers a CFO (Cognitive Flow Optimizer) refinement loop.

## 3. The Synergistic Iterative Cycle

This framework operates as a continuous, multi-stage pipeline designed for **perpetual coherence**:

1.  **Ingestion & Contextualization:** As new data enters the system, the **ESF** attaches Episemantic Markers, grounding the data in its exact temporal, structural, and relational context.
2.  **Continuous Epistemic Monitoring:** The **Truth Compass** monitors the newly ingested data against the global baseline. If the data introduces cognitive dissonance or contradicts existing Prime Truths, the compass needle drifts, generating an alert.
3.  **Stress Testing (The Crucible):** The **Tester (Active Validation)** takes the annotated, monitored data and runs simulations. It attempts to break the newly formed semantic links to ensure they do not cause infinite recursion or structural decay.
4.  **Auditing & Canonization:** The **Judge (Compliance Enforcement)** reviews the output. It verifies that all OGLN standards, Sovereignty Metadata Locks (UAM), and Prime Directives are respected.
5.  **Refinement:** If the Judge or Tester flags an anomaly, the data is rejected, and a `[TASK_RESULT_PACKET]` is generated. This failure is fed back into the system ("Failure as Data") to refine the ESF's marker application rules and update the system's active learning models.
