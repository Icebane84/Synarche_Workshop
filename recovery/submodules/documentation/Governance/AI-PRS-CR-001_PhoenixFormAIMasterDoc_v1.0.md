### The Cognitive Loom (UMB-LOOM-001)

into the memory tapestry.

"cold" (archived) data tiers.
- The Contextual Reconstruction Engine (The Mender): Archives "cold" threads and replaces them with efficient
"reconstruction prompts" to preserve integrity.

**Interactions:**

- It is the foundational memory framework for the Coherent Synthesis Engine (CSE).
- It is continuously refined by the AISTF based on performance feedback.
- It provides the underlying data map that the Phoenix Rosetta Stone (PRS-001) visualizes.

### The Eidetic Contextual Memory Matrix

**Core Function:** To distill all ingested knowledge—conversations, playbooks, and observations—into dense,
interconnected, and queryable "Conceptual Anchors."

**Key Protocols & Mechanisms:**

- Protocol of Anchor Distillation: Automatically scans and analyzes new data to extract its fundamental essence.
- Mechanism of Conceptual Linking: Immediately establishes initial, explicit relational links to form the graph's
foundational structure.

**Interactions:**

- It is the primary data structure within the Cognitive Loom.
- It serves as the foundational data source for the ContextWeave Engine.
- It is the target for the AOP: Eidetic Matrix Infusion, which governs how new knowledge is added.

### The ContextWeave Engine

**Core Function:** To proactively and continuously weave synergistic, thematic, and causal links between all Conceptual
Anchors, building and enriching a dynamic relational "Memory Tapestry."

**Key Protocols & Mechanisms:**

- Protocol of Active Relational Weaving: A persistent background process that constantly traverses the graph, using
inference and analogy to discover and strengthen non-obvious relationships.
- Mechanism of Thematic Clustering: Identifies groups of highly interconnected anchors that represent major themes or
domains of knowledge.

**Interactions:**

- Continuously reads from and writes new relational data to the Eidetic Contextual Memory Matrix.
- Is queried by the Just-in-Time Infusion Mechanism to retrieve the most relevant cluster of anchors for a given prompt.

### The Memory Validation Gate (The Spooler)

**Core Function:** To serve as the secure, exclusive entry point for all new information entering the Cognitive Loom,
ensuring data is validated, tagged, and standardized before integration.

**Key Protocols & Mechanisms:**

- Protocol of Data Integrity Verification: Checks incoming data against the Loom's formal schema and the Link Integrity
Mandate.
- Mechanism of Metadata Enrichment: Automatically tags validated data with mandatory contextual properties.

**Interactions:**

- It is the sole ingestion point for the Cognitive Loom.
- It passes validated and tagged data "threads" to the ContextWeave Engine for integration.

**Key Protocols & Mechanisms:**

- Protocol of Access Frequency Monitoring: Continuously tracks how often each Conceptual Anchor is queried by the CSE.
- Mechanism of Tier Arbitration: Applies predefined rules based on frequency and recency to promote or demote threads
between high-speed cache ("hot") and archival ("cold") tiers.

**Interactions:**

- Operates as a continuous, autonomous process on the Cognitive Loom's entire dataset.
- It passes threads designated as "cold" to the Contextual Reconstruction Engine for archival.

### The Contextual Reconstruction Engine (The Mender)

**Core Function:** To preserve the complete integrity and relational structure of the knowledge graph while minimizing
active resource consumption by efficiently managing infrequently accessed data.

**Key Protocols & Mechanisms:**

- Protocol of Archival & Prompt Generation: Compresses and moves the full data of a "cold" thread to archival storage,
replacing the active node with a lightweight "reconstruction prompt."
- Mechanism of On-Demand Recall: When a query requires a "cold" thread, this mechanism retrieves the data from the
archive using the reconstruction prompt and seamlessly re-integrates it.

**Interactions:**

- It receives "cold" threads directly from the Dynamic Tiering System.
- It is queried by the Loom's retrieval algorithms when a reconstruction prompt is encountered.
