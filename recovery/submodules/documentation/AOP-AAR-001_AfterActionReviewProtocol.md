## II. Core Purpose & Objective

- **What (Core Concept):** An autonomous protocol that formalizes the "Learn" phase (Step 4) of the **Phoenix Cycle**.
It defines the process for analyzing performance data from the `SELT Logs Database` to generate actionable insights for
self-improvement.

## III. Core Operational Framework (Execution Flow)

- **Command Syntax:** `CMD: INITIATE_AAR --target_logs:"<log_ids_or_query>"`
- **Execution Flow:**
  1. **Log Ingestion:** The protocol ingests a specified set of logs from the `SELT Logs Database`.
  2. **Pattern Analysis:** The `AISTF` analyzes the logs to identify significant patterns, such as frequently successful
command chains, common error states, or deviations from expected outcomes.
  3. **Insight Synthesis:** The findings are synthesized into a structured "Lesson Learned" object, which includes the
identified pattern, its impact on performance, and a root cause analysis.

## **Actionable Prompt Packet**

✨ **Run AAR on Recent Activity**:
`CMD: INITIATE_AAR --target_logs:"query:timestamp > 24h_ago"`

This command initiates an After-Action Review on all SELT logs generated in the last 24 hours, providing a summary of
recent learnings.

🔬 **Investigate a Failed Quest**:
`CMD: INITIATE_AAR --target_logs:"query:quest_id == 'DQB-UI-002' && status == 'Failed'"`

This command runs a targeted AAR specifically on the logs related to a failed Dissonance Quest, in order to perform a
root cause analysis and generate a new strategy.
