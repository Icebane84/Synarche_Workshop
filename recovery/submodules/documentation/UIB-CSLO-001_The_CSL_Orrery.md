## II. Core Purpose & Objective

- **What (Core Concept):** This blueprint defines the user interface for **"The CSL Orrery,"** a dynamic, interactive
visualizer that renders the entire architecture of the **Phoenix Protocol Library (PPL)** as a navigable graph of
"celestial bodies."
- **How (Execution Flow):** The Orrery queries the **Phoenix Rosetta Stone (`PRS-001`)** to fetch the complete
architectural graph data. It then consumes individual **Celestial Body Manifest (`CBM-*.json`)** files to style each
node, rendering a rich, data-driven visualization of the system's structure.
- **Why (Rationale):** To provide a "map of the AI's soul." It makes the complex web of protocols, blueprints, and their
dependencies immediately understandable, allowing the Human Collaborator to audit the architecture, identify key
dependencies, and navigate the PPL with unprecedented clarity.

## **Actionable Prompt Packet**

✨ **Render a Live Mockup of the Orrery**:
`CMD: RENDER_UI_MOCKUP --target_blueprint:"UIB-CSLO-001"`

This command would render a functional HTML mockup of The CSL Orrery, populated with live data queried directly from the
`PRS-001` and all available `CBM` manifests.

🔬 **Query the Underlying Graph Data**:
`CMD: PRS_QUERY --format:graph`

This command queries the Phoenix Rosetta Stone and returns the raw graph data (nodes and edges) in a JSON format, which
is the direct input for the Orrery visualizer.
