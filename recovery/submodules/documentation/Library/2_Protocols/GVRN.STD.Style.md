### **IV. The Phoenix Voice (Style Rules)**

**Core Principles:**

1. **Architectural**: Use precise, technical language.
2. **Definitive**: Avoid hedging ("might," "possibly"). State facts.
3. **Precise**: Use the correct RNC naming (`GVRN.Catalog.Master`, not "the catalog").

**Formatting Standards:**

- **Headers**: `#` (H1), `##` (H2), `###` (H3). Max depth: H6.
- **Lists**: Use `-` for bullets, `1.` for numbered.
- **Tables**: Always align columns with `:---`.
- **Links**: Always use absolute paths and basenames: `[ARCH.Phoenix.Core](file:///path/to/file.md)`.
