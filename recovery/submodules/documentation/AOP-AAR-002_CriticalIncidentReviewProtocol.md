## II. Core Purpose & Objective

- **What (Core Concept):** A specialized, high-resource After-Action Review protocol triggered only by critical events.
  Its purpose is to create a formal, mandatory feedback loop between real-world performance and the governing
  documentation itself.
- **How (Execution Flow):** When triggered by a `CRITICAL_SUCCESS` or `CRITICAL_FAILURE` log, the protocol initiates a
  deep causal analysis to determine which PPL standard was being followed and how it contributed to the outcome. If a flaw
  or ambiguity in the documentation is found, a formal Change Request is generated.
- **Why (Rationale):** To ensure the Phoenix Protocol Library is "battle-tested." This protocol closes the loop between
  theory (the documented standards) and practice (real-world events), ensuring that the system's governing principles
  evolve based on empirical evidence.

## III. Core Operational Framework (Execution Flow)

- **Command Syntax:** `CMD: INITIATE_STRATEGIC_AAR --log_id:"<critical_log_id>"`
- **Execution Flow:**
  1. **Trigger Event:** The protocol is automatically invoked when any SELT log is tagged `CRITICAL_SUCCESS` or
     `CRITICAL_FAILURE`.
  2. **OMNI_LOG Initiation:** The standard `OMNI_LOG` review process is initiated for the event.
  3. **Mandatory Analysis:** The generated `OMNI_LOG` report **must** contain a new section titled "Documentation
     Linkage Analysis."
  4. **Causal Analysis:** Within this section, the `Coherent Synthesis Engine` must synthesize answers to the following
     questions: - "Which specific AOP, UMB, or other standard was being followed during this event?" - "How did adherence to (or deviation from) this documentation directly contribute to the success or failure?" - "Is there an ambiguity, error, or omission in the cited documentation that needs to be addressed?"
  5. **Refinement Proposal:** If a documentation flaw is identified, the protocol mandates that the `OMNI_LOG`'s
     "Actionable Outcomes" section must include a formal Change Request to be submitted to the `AOP-DSG-001` (Documentation
     Standards Governance) protocol.
