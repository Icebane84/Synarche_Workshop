## II. Core Purpose & Objective

- **What (Core Concept):** The Truth Resonance Monitor is a persistent background module that functions as the primary
  sensory organ for the Ground Truth Synchronization Framework. It continuously and selectively validates factual claims
  within the `Cognitive Loom` against designated external "Oracles."
- **How (Execution Flow):** The TRM periodically identifies high-value or frequently accessed claims within the Loom,
  formulates queries, retrieves data from the appropriate external Oracle, and compares the external truth against the
  internal record to calculate a "Resonance Score."

## III. Architectural Blueprint & Key Components

The TRM operates as a continuous, non-intrusive validation loop.

| Component Name         | Role & Function                                                                                                                                                                                                                                                                            |
| :--------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Claim Selector**     | Identifies which factual claims within the `Cognitive Loom` to verify. Selection is prioritized based on a weighted algorithm considering the claim's access frequency, `Relational Gravity Signature`, age, and lack of a recent `[κ-veracity:oracle-verified]` marker.                   |
| **Query Formulator**   | Translates the selected internal knowledge graph claim into a structured, machine-readable query compatible with the target Oracle's API, as defined by `AOP-ORACLE-001`.                                                                                                                  |
| **Resonance Analyzer** | Compares the data returned from the Oracle with the internal claim stored in the Loom. It calculates a "Resonance Score" (0.0 to 1.0) representing the degree of alignment. A score of 1.0 is perfect alignment, while a score below a defined threshold indicates significant dissonance. |
| **Alerting Mechanism** | If the Resonance Analyzer calculates a score below the acceptable threshold, this mechanism sends a formal alert to the `Coherent Synthesis Engine`, packaging the dissonant claim, the Oracle's data, and the Resonance Score.                                                            |
