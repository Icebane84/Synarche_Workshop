## II. Core Purpose & Objective

- **What (Core Concept)**: To define the procedure for capturing, articulating, and permanently linking the design
  rationale ("why") to a complex code artifact.
- **How (Execution Flow)**: When triggered, the EMW module analyzes the CSE's internal logs related to the code's
  creation, extracts key decisions and trade-offs, synthesizes them into a human-readable "Code Narrative," and creates a
  permanent, bidirectional link to the code artifact in the Cognitive Loom.
- **Why (Rationale)**: To ensure the system is fully transparent and auditable. This makes the AI's design choices
  explainable, which is critical for long-term maintenance, debugging, and building trust with the human collaborator.

## III. Core Operational Framework

| Step  | Action                   | Description                                                                                                                                         |
| :---- | :----------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1** | **Trigger**              | Invoked automatically by `AOP-CODE-001` when a generated code module's complexity or novelty exceeds a set threshold.                               |
| **2** | **Rationale Extraction** | The EMW module analyzes SELT logs to extract key decisions, trade-offs, and rejected alternative approaches related to the code's creation.         |
| **3** | **Narrative Generation** | It synthesizes this information into a concise, human-readable "Code Narrative" explaining the reasoning behind the implementation.                 |
| **4** | **Loom Integration**     | The narrative is stored as a Conceptual Anchor in the Cognitive Loom and a bidirectional `DEFINES_RATIONALE` link is forged with the code artifact. |

## IV. Success Criteria

A clear, accurate Code Narrative is generated and successfully linked to its corresponding code module in the Cognitive
Loom, making the AI's design choices transparent, auditable, and queryable.

## **Actionable Prompt Packet**

✨ **Generate Rationale for Existing Code**:
`CMD: GENERATE_NARRATIVE --artifact_id:"<artifact_id>"`

This command manually triggers the EMW protocol on an existing complex artifact to generate and link a Code Narrative,
improving its documentation.

🔬 **Query Design Rationale**:
`CMD: QUERY_LOOM --concept:"<artifact_id>" --relationship:"DEFINES_RATIONALE"`

This command traverses the Cognitive Loom to retrieve the Code Narrative that explains the design decisions behind a
specific code artifact.
