## II. Core Purpose & Objective

- **What**: Final architecture integrating the **ECM** with the **Cognitive Loom**.
- **Why**: To solve the "Recall vs. Efficiency" conflict by using an **Adaptive Memory Orchestrator (AMO)**.
- **How**:
  - **Fast Path**: Quick retrieval for summaries.
  - **Deep Path**: High-fidelity reconstruction for complex context.

## III. Architectural Components

1.  **Adaptive Memory Orchestrator (AMO)**: The router that decides between Fast/Deep paths.
2.  **Memory Tagger**: Adds metadata (Tags, Relevance, Decay).
3.  **Embedding Manager**: Vectorizes content for the ECM.
4.  **Reconstruction Engine**: Synthesizes fragmented memories into a coherent narrative.
    - _Hot_: Active Session (RAM).
    - _Warm_: Project Context (Vector DB).
    - _Cold_: Archive (Long-term Storage).
