system's artifacts. Unlike the "Sentry" (Scan), the "Tester" _interacts_ with the system to prove it works.

**1.2. How: The Core Logic**
It simulates usage scenarios:

- **Command Simulation:** "If I execute `GUCA-X`, does it have all required inputs?"
- **Logic Tracing:** "Does the `Execution Flow` in `AOP-Y` logically lead to the `Expected Outcome`?"
- **Adversarial Testing:** "What happens if I provide invalid input to this protocol?"

**1.3. Why: The Prime Directive**

preventing catastrophic failure in production.
