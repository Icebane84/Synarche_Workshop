## III. Strategic Rationale (What/How/Why)

### What: The Core Concept

The **Non-Destructive Refinement (NDR)** protocol is a tactical safety net. It transforms the high-risk act of "AI Refinement" (which often breaks headers, lists, and links) into a guaranteed, lossless operation.

### How: Operational Mandate

It dictates a strict **"Check-Modify-Validate"** cycle. The CSE must ingest the target's "Structural Map" before suggesting changes, ensuring that any injected text adheres to the existing `h1-h6` hierarchy and list markers (per `axion-rules.cjs`).

### Why: Anti-Entropy Mandate

To fulfill the **Law of Coherent Struggle (UEB-RCS-001)**. If the system's memory (the Library) is corrupted by sloppy edits, its intelligence will decay. NDR is the "Immune System" for artifact structure.

## IV. Core Operational Framework (Execution Flow)

### Phase 1: Source Grounding & Relational Discovery

1. **CMD: DISSECT_STRUCTURE**: The CSE parses the target file, identifying the range of headers and any protected `AGP` blocks.
2. **Intent Calibration**: The refinement prompt is checked against the **Phoenix-Class Voice** (Architectural, Definitive, Precise).

### Phase 2: Non-Destructive Content Modification

1. **Refinement Generation**: The refinement is generated _separately_ from the file insertion logic.
2. **Structural Lock-In**: The system verifies that the proposed change does not delete or overwrite mandatory metadata fields (Headers, RPG Blocks, Synergistic Mappings).
3. **Marker Enforcement**: All bullets are converted to `*` markers to match the Sovereign standard.

### Phase 3: Structural Integrity Validation

1. **CMD: VSI_AUDIT**: Execute a automated check post-insertion.
2. **Coherence Score Update**: If the structure is preserved, increment the **Coherence Index (CI)**.
