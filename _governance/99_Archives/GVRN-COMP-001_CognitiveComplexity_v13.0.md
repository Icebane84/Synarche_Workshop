## II. Objective

To move beyond Cyclomatic Complexity (testability) and enforce **Cognitive Complexity** (understandability). High scores indicate code that is "difficult to reason about," representing a violation of the **Zero Entropy Rule**.

## III. The Lexical Rules (The Sonar Method)

### 3.1. Rule 1: Reward Readable Shorthand

Ignore structures that condense implementation detail into intuitive calls.

- **No Increment**: Method definitions, null-coalescing (`??`), or declarative class definitions.

### 3.2. Rule 2: Penalize Breaks in Linear Flow

Incremental penalties (+1) for every point where the brain must work to track state or path changes.

- **Increment For**:
  - **Loops**: `for`, `while`, `do while`.
  - **Conditionals**: `if`, `else if`, `else`, ternary operators.
  - **Control Jumps**: `break LABEL`, `continue LABEL`.
  - **Error Handling**: `catch` (Note: `try` and `finally` are free).
  - **Logic**: Each sequence of binary logical operators (`&&`, `||`).
  - **Recursion**: Each method in a recursion cycle.

### 3.3. Rule 3: The Nesting Multiplier

Structures nested inside others receive an additional penalty commensurate with their depth.

- **Formula**: `Base Increment + Nesting Level`
- _Example_: An `if` (1) inside a `for` (1) inside a `while` (1).
  - While = +1
  - For = +2 (1 base + 1 depth)
  - If = +3 (1 base + 2 depth)
  - **Total = 6**
