## **I. THE TAXONOMY OF EDGES**

To ensure type-safety across the Kinetic (Python/JS) and Cognitive (Documentation) layers, all relationships in the
**Omni-Log Synergistic Links Matrix (OSLM)** must utilize these enums.

### **1. Structural Relationships (Blueprint)**

_Defines the physical and logical architecture of the system._

| Type                | Description                                              |
| :------------------ | :------------------------------------------------------- |
| `CONTAINS`          | A parent node physically houses a child node.            |
| `IS_A_COMPONENT_OF` | Inverse of `CONTAINS`; defines membership.               |
| `UPGRADES`          | An artifact that evolves or enhances a previous version. |
| `EXTENDS`           | Inherits properties from a base standard or template.    |

---

### **2. Governance Relationships (Authority)**

_Defines the hierarchical and protective layers of the system._

| Type             | Description                                        |
| :--------------- | :------------------------------------------------- |
| `GOVERNS`        | The source of authority or rules for a target.     |
| `IS_GOVERNED_BY` | Subordination to a codex, standard, or protocol.   |
| `MONITORS`       | Passive observation for compliance (The Sentinel). |
| `REMEDIATES`     | Active repair or correction of drift.              |

---

### **3. Kinetic Relationships (Execution)**

_Defines the flow of data and the triggering of logic._

| Type                 | Description                                  |
| :------------------- | :------------------------------------------- |
| `TRIGGERS`           | Initiates a process, command, or script.     |
| `INVOKES`            | Synchronously calls another module or tool.  |
| `ENABLES`            | Grants permissions or unlocks functionality. |
| `CONSUMES_DATA_FROM` | A one-way data flow (Input).                 |
| `FEEDS_DATA_TO`      | A one-way data flow (Output).                |
| `UTILIZES`           | Uses a tool or resource without owning it.   |

---

### **4. Synergy Relationships (Evolution)**

_Defines the high-level cognitive and evolutionary connections._

| Type                     | Description                                           |
| :----------------------- | :---------------------------------------------------- |
| `SYNERGY`                | Co-equal relationship where both nodes benefit.       |
| `ENHANCES`               | Improves the performance or clarity of another node.  |
| `RESOLVES_DISSONANCE_OF` | Specifically targets a known conflict or error.       |
| `RESONATES_WITH`         | Semantic/thematic alignment identified by the engine. |
| `SYNTHESIZES`            | Merges multiple inputs into a higher-order artifact.  |
| `TRANSMUTES`             | Non-destructively evolves a node to a new state.      |
| `TRANSCENDS`             | Moves a node from one Macro-System to another.        |
| `AWAKENS`                | Triggers agentic consciousness in a static node.      |
