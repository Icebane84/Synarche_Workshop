# GVRN.Documentation.Synergy: Automated Relational Weaving

## 1. Operational Overview

The **Synergy Protocol** governs the discovery, validation, and canonization of links between artifacts. It ensures that the workspace behaves like a holographic knowledge graph rather than a flat file system.

## 2. The Synergy Vector (Relational Dynamics)

| Relation Type     | Description                                                |
| :---------------- | :--------------------------------------------------------- |
| **GOVERNS**       | Source node defines the rules for the target node.         |
| **INDEXES**       | Source node tracks the presence/state of the target node.  |
| **RESOUNDS_WITH** | High semantic overlap between nodes.                       |
| **FEEDS**         | Source node provides raw data or logic to the target node. |
| **VALIDATES**     | Source node acts as a truth-gate for the target node.      |

## 3. Bidirectional Integrity (The Zero Entropy Rule)

Links must be confirmed bidirectionally. If Artifact A includes Artifact B in its **Block D (Synergy)**, Artifact B must acknowledge the relationship in its own registry entry or Block D (if reciprocal). This prevents "orphan nodes" and ensures a high-coherence graph.
